## 🎯 部署组件说明

### 1. Web 应用（Cloudflare Pages）

#### 前端部分
- **技术栈**: React 18 + TypeScript + Vite
- **部署位置**: Cloudflare Pages（全球 CDN）
- **构建产物**: 静态 HTML/CSS/JS 文件
- **访问方式**: `https://your-project.pages.dev` 或自定义域名

#### 后端部分（Pages Functions）
- **技术栈**: Cloudflare Workers（V8 JavaScript Runtime）
- **部署位置**: Cloudflare 边缘节点（全球分布）
- **功能**: RESTful API、认证、数据库操作
- **路由**: `/api/*` 路径下的所有请求

#### 数据存储
- **D1 数据库**: SQLite 兼容的关系型数据库
  - 存储用户、书签、标签、标签页组等数据
  - 支持 SQL 查询和事务
  
- **KV 存储**: 键值对存储
  - `RATE_LIMIT_KV`: 存储 API 限流数据
  - `PUBLIC_SHARE_KV`: 存储公开分享缓存

### 2. 浏览器扩展（独立分发）

#### 扩展组件
- **Popup**: 主界面，保存书签和收纳标签页
- **Options**: 配置页面，设置 AI 和 API
- **Background**: 后台服务，同步和通知
- **Content Script**: 页面信息抓取

#### 本地存储
- **IndexedDB (Dexie)**: 离线缓存书签和标签
- **Chrome Storage**: 配置信息和用户偏好

#### 分发方式
- **开发者模式**: 从网站下载 zip，手动加载
- **Chrome Web Store**: 官方商店发布（可选）

## 🔄 数据流向

### 用户保存书签流程

```
1. 用户在网页点击扩展图标
   ↓
2. Content Script 抓取页面信息（标题、URL、描述）
   ↓
3. 发送到 Popup 界面
   ↓
4. Popup 调用 AI API（OpenAI/Anthropic 等）生成标签
   ↓
5. 用户选择/编辑标签，点击保存
   ↓
6. 扩展发送 POST 请求到 Cloudflare Pages API
   ↓
7. Pages Functions 验证 API Key
   ↓
8. 写入 D1 数据库
   ↓
9. 返回成功响应
   ↓
10. 扩展更新本地 IndexedDB 缓存
    ↓
11. 显示成功提示
```

### 用户浏览书签流程

```
1. 用户访问 Web 应用
   ↓
2. React 应用加载（从 Cloudflare CDN）
   ↓
3. 用户登录（JWT 认证）
   ↓
4. 前端发送 GET 请求到 /api/v1/bookmarks
   ↓
5. Pages Functions 验证 JWT Token
   ↓
6. 查询 D1 数据库
   ↓
7. 返回书签列表（JSON）
   ↓
8. React 渲染书签卡片
   ↓
9. 用户可以筛选、搜索、编辑
```

### 公开分享流程

```
1. 用户在 Web 应用启用公开分享
   ↓
2. 系统生成唯一 slug（如 /share/john-bookmarks）
   ↓
3. 用户设置公开/私密书签
   ↓
4. 访客访问分享链接
   ↓
5. Pages Functions 查询 PUBLIC_SHARE_KV 缓存
   ↓
6. 如果缓存未命中，查询 D1 数据库
   ↓
7. 返回公开书签列表
   ↓
8. 写入 KV 缓存（TTL: 5分钟）
   ↓
9. React 渲染公开分享页面
```

## 🌍 部署位置

### Cloudflare 全球网络

TMarks 部署在 Cloudflare 的全球边缘网络上，包括：

- **北美**: 美国、加拿大、墨西哥
- **欧洲**: 英国、德国、法国、荷兰等
- **亚洲**: 日本、新加坡、香港、台湾、韩国
- **大洋洲**: 澳大利亚、新西兰

**优势**:
- 低延迟：用户请求由最近的边缘节点处理
- 高可用：自动故障转移和负载均衡
- 免费 SSL：自动 HTTPS 证书
- DDoS 防护：内置安全防护

## 🔐 安全架构

### 认证和授权

```
┌─────────────────────────────────────────────────────────────┐
│                      认证流程                                 │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  用户登录                                                     │
│    ↓                                                          │
│  验证用户名/密码（bcrypt）                                     │
│    ↓                                                          │
│  生成 JWT Token                                               │
│    ├─ Access Token (短期，如 15 分钟)                        │
│    └─ Refresh Token (长期，如 7 天)                          │
│    ↓                                                          │
│  返回给客户端                                                 │
│    ↓                                                          │
│  客户端存储在 localStorage                                    │
│    ↓                                                          │
│  后续请求携带 Access Token                                    │
│    ↓                                                          │
│  服务端验证 Token                                             │
│    ├─ 有效：处理请求                                          │
│    └─ 过期：使用 Refresh Token 刷新                          │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### API Key 认证（扩展使用）

```
┌─────────────────────────────────────────────────────────────┐
│                    API Key 流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  用户在 Web 应用创建 API Key                                  │
│    ↓                                                          │
│  生成随机密钥（32 字节）                                       │
│    ↓                                                          │
│  存储 Hash 到数据库                                           │
│    ↓                                                          │
│  返回明文密钥给用户（仅一次）                                  │
│    ↓                                                          │
│  用户配置到浏览器扩展                                          │
│    ↓                                                          │
│  扩展请求携带 X-API-Key Header                                │
│    ↓                                                          │
│  服务端验证 API Key                                           │
│    ├─ 有效：处理请求                                          │
│    ├─ 无效：返回 401                                          │
│    └─ 过期：返回 401                                          │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

### 数据加密

- **传输加密**: 全站 HTTPS（TLS 1.3）
- **密码加密**: bcrypt（成本因子 10）
- **敏感数据**: AES-256 加密存储
- **API Key**: SHA-256 哈希存储

## 📈 性能优化

### 前端优化

- **代码分割**: React.lazy + Suspense
- **懒加载**: 路由级别的代码分割
- **压缩**: Gzip + Brotli
- **缓存**: Service Worker（可选）
- **CDN**: Cloudflare 全球 CDN

### 后端优化

- **数据库索引**: 关键字段建立索引
- **KV 缓存**: 热点数据缓存
- **查询优化**: 避免 N+1 查询
- **连接池**: D1 自动管理
- **边缘计算**: 请求在边缘节点处理

### 扩展优化

- **本地缓存**: IndexedDB 离线存储
- **防抖**: 搜索和标签选择防抖
- **批量操作**: 减少 API 调用次数
- **增量同步**: 只同步变更数据

## 🔄 更新和维护

### Web 应用更新

```bash
# 1. 修改代码
# 2. 提交到 Git
git add .
git commit -m "Update feature"
git push origin main

# 3. Cloudflare Pages 自动部署
# 4. 通常 1-3 分钟完成
```

### 浏览器扩展更新

```bash
# 1. 修改扩展代码
# 2. 更新版本号（manifest.json）
# 3. 构建新版本
cd tab
pnpm build

# 4. 打包扩展
# Linux/Mac: cd dist && zip -r ../../tmarks/public/tmarks-extension.zip . && cd ../..
# Windows: Compress-Archive -Path "dist\*" -DestinationPath "..\tmarks\public\tmarks-extension.zip" -Force

# 5. 更新下载页面
# 6. 提交并推送
git add .
git commit -m "Update extension v1.0.1"
git push origin main

# 7. 如果发布到商店，上传新版本
```

### 数据库迁移

```bash
# 1. 创建迁移文件
# 2. 测试迁移（本地）
pnpm db:migrate:local

# 3. 应用到生产
pnpm db:migrate

# 4. 验证数据
wrangler d1 execute tmarks-prod-db --command "SELECT * FROM schema_migrations;"
```

## 🎯 总结

TMarks 采用现代化的 Serverless 架构：

✅ **前端**: React SPA，部署在 Cloudflare CDN  
✅ **后端**: Cloudflare Workers，全球边缘计算  
✅ **数据库**: D1（SQLite）+ KV 存储  
✅ **扩展**: 独立构建，多种分发方式  

**优势**:
- 🚀 性能出色（边缘计算 + CDN）
- 💰 成本低廉（免费额度充足）
- 🌍 全球可用（自动多地域部署）
- 🔒 安全可靠（内置 DDoS 防护）
- 📈 易于扩展（自动伸缩）

**适用场景**:
- 个人书签管理
- 团队知识库
- 公开书签分享
- 浏览器历史管理
